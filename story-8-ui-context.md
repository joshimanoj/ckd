## 2. Relevant User Flows

From Design.md §4 — Flow 3: Parent Checks Watch Time

**Flow 3: Parent Checks Watch Time**
1. Video Library → parent taps parent icon (top-right) → Parental Gate modal appears
2. Parent solves gate (single-digit addition/subtraction) → gate dismisses
3. Parent Panel side drawer slides in from right (web)
4. Dashboard tab shown → useDashboard fires 3 Firestore queries
5. While loading: shimmer skeleton
6. On data: today total + weekly bar chart + monthly total
7. Parent reads data → clicks close button or clicks outside drawer
8. Drawer slides out → returns to Library

**Success state:** Parent sees "45 min" (or other formatted value) in large purple text.
**Error state:** "Couldn't load data. Try again." + retry button.
**Empty state:** "No watch time recorded yet. Start a video to begin tracking."

## 4. Interaction Rules

From Design.md §7:

| Rule | Text | Applies to |
|------|------|------------|
| Loading states | "Dashboard data: shimmer skeleton bars — never a standalone full-screen spinner — always skeleton in context" | DashboardScreen (loading) |

| Error states | "Network error: ... + Retry button (replaces content area, not a modal)" | DashboardScreen (error) |

| Parent Panel dismiss | "Swipe down to dismiss: bottom sheet (Parent Panel)" [web: click outside] | ParentPanel overlay click |
| Bottom sheet | "Bottom sheet sliding up ... over Library" [web: side drawer replaces] | ParentPanel container |
