# UAT Checklist: Story #1 — Google Sign-In & Authentication

Generated by /prd | Sources: story-1.md (acceptance criteria, user flows), story-1-ui-context.md (components, tokens)
Pass: 1 — Web | Type: UI

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
Firebase emulator must be running: `firebase emulators:start`

**Test data:**
- Use a real Google account to test the actual popup flow
- For routing tests: use Firebase Auth emulator REST API to seed users with specific `consentGiven` states

**Start at:** A fresh incognito browser window — no cached Firebase session

**Required browser:** Chrome (primary), Samsung Internet (secondary for India target)
**Required viewport:** Start at 390×844 (iPhone 14 equivalent — represents mobile web target)

---

## Checklist

### Flow 1: First-Time Sign-In (Happy Path)

- [ ] 1. Open `http://localhost:5173` in incognito — Sign In screen appears
- [ ] 2. Upper half has brand gradient (coral-red → purple → pink). Lower half is near-white `#FAFAFA`
- [ ] 3. Creator photo is visible: circular, 120px, gold ring border. Prominently centred.
- [ ] 4. App name "Choti Ki Duniya" appears below photo in brand purple, Baloo 2 ExtraBold
- [ ] 5. "Continue with Google" button is coral-red, pill shape, full-width (within max-width container), minimum 48px height
- [ ] 6. Tap "Continue with Google" — Google account picker popup opens
- [ ] 7. Select a Google account — popup closes, app transitions away from Sign In screen
- [ ] 8. Routed to Consent screen (stub: `/consent`) — Sign In screen no longer visible
- [ ] 9. Open Firebase Emulator UI → Firestore → users/{uid}: document exists with `consentGiven: false`, `notificationsEnabled: false`, `fcmToken: null`
- [ ] 10. `createdAt` field is present and is a Timestamp

### Flow 2: Return Visit (Already Signed In, Consent Given, Child Profile Exists)

- [ ] 1. Seed emulator: user with `consentGiven: true`, child profile doc exists in subcollection
- [ ] 2. Open `http://localhost:5173` (not incognito — reuse session, or sign in via emulator)
- [ ] 3. Sign In screen never appears — routed directly to `/library`

### Flow 3: Return Visit (Consent Given, No Child Profile)

- [ ] 1. Seed emulator: user with `consentGiven: true`, no child profiles
- [ ] 2. Open app — routed directly to `/profile` (stub screen visible)
- [ ] 3. Sign In screen never appears

### Flow 4: Return Visit (Mid-Consent Kill — consentGiven: false)

- [ ] 1. Seed emulator: authenticated user with `consentGiven: false`
- [ ] 2. Open app — routed to `/consent` (not to Sign In, not to Library)
- [ ] 3. Correct behaviour: app remembers incomplete consent state

### Flow 5: Sign-Out

- [ ] 1. While signed in (any state), trigger sign-out (call `window.__signOut()` in DevTools console, or use the test hook)
- [ ] 2. App returns to Sign In screen at `/`
- [ ] 3. Refreshing the page keeps you on Sign In (session cleared)

### Flow 6: Error — Network Failure During Sign-In

- [ ] 1. Open app → Sign In screen visible
- [ ] 2. Simulate network failure: DevTools → Network tab → Offline, then click "Continue with Google"
- [ ] 3. Error message appears below button: "Sign in failed. Please try again."
- [ ] 4. Error text is red (`#EF4444`), 13sp
- [ ] 5. Sign In screen is still fully visible — no crash, no blank screen
- [ ] 6. Button is re-enabled (can retry)

### Flow 7: User Cancels Google Account Picker

- [ ] 1. Click "Continue with Google" — picker opens
- [ ] 2. Close the picker (click outside / press Escape)
- [ ] 3. No error message shown
- [ ] 4. Sign In screen remains, button re-enabled silently

### Flow 8: Offline on Launch

- [ ] 1. Set DevTools → Network → Offline BEFORE loading the page
- [ ] 2. Open `http://localhost:5173`
- [ ] 3. Offline screen shown — "Check your connection" message + Retry button visible
- [ ] 4. The "Continue with Google" button is NOT visible
- [ ] 5. Restore network → click Retry → Sign In screen appears (or auto-restores on `online` event)

## Visual Check

- [ ] Brand gradient (`#F43F5E → #9333EA → #EC4899`, 135deg) covers upper half of Sign In screen
- [ ] Creator photo: circular, 120px, gold ring border — visually prominent
- [ ] App name: Baloo 2 ExtraBold, ~28px, brand purple `#9333EA`
- [ ] Button: coral-red `#F43F5E`, pill shape (24px radius), white label in Nunito SemiBold, ≥48px height
- [ ] Error text: red `#EF4444`, 13px Nunito, inline below button
- [ ] LoadingSpinner (while auth resolves): brand purple spinner centred full-screen — visible for < 1 second on fast connection

---

## Responsive / Device Check

- [ ] 375px viewport (small phone): all elements visible, no horizontal scroll, button not cut off
- [ ] 480px viewport: Sign In screen renders correctly, 2-column grid not involved (single column auth screen)
- [ ] 768px viewport: Sign In screen centred with max-width constraint, no layout breakage
- [ ] No horizontal scrollbar at any width ≥ 320px
- [ ] Button touch target ≥ 48px height (use DevTools → inspect box model to verify)

---

## Edge Cases

- [ ] Reload page while signed in with `consentGiven: true` + profile: never flashes Sign In screen (loading spinner shown during resolution, then direct to library)
- [ ] `consentGiven` field is exactly `false` (boolean, not `null` or `undefined`) in Firestore for new users
- [ ] No Firebase Analytics events in network tab during any part of the auth flow
- [ ] No advertising ID accessed (no `IDFA`, `GAID` network calls in emulator network log)
