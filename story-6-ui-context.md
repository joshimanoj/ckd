## 2. Relevant User Flows

### Flow 5: Admin Publishes a Video
Source: Design.md §4, Flow 5

1. Admin navigates to `/admin` (protected — admin claim required)
2. Sees video list + "Add Video" button → clicks "Add Video"
3. Form: YouTube Video ID, Title, Category (dropdown), Duration (mm:ss),
   Thumbnail URL (auto-populated or custom)
4. Clicks "Publish" → video appears in Firestore → immediately visible in app
5. Admin optionally fills Notification panel → clicks "Send" → writes to
   `notifications` collection → Cloud Function sends FCM to opted-in users

**Success state:** Video appears in list within seconds of publishing.
**Error state:** Invalid YouTube ID → inline validation error before submit.

### Access Control Flow

1. Unauthenticated user hits `/admin` → silent redirect to `/`
2. Authenticated non-admin hits `/admin` → silent redirect to `/` (no error shown)
3. Admin hits `/admin` → `getIdTokenResult(user, forceRefresh: true)` checks claim
   → admin panel renders

## 4. Interaction Rules

### Form Validation (Task A-7)
Source: Design.md §7 — Form patterns

- Inline validation fires **on blur**, not on keystroke
- Required fields marked with asterisk
- Error text inline below field in red (`#EF4444`), 13sp
- Applies to: VideoForm fields (youtubeVideoId, title, duration)

### Button Disabled State (Task A-7)
Source: story-6.md Acceptance Criteria (Non-Functional)

- "Publish" button disabled while Firestore write is in progress
- Prevents double-submit
- Visual: reduced opacity, cursor: not-allowed

### Success Toast (Task A-9)
Source: Design.md §7 — Feedback

- Slides up from bottom
- Duration: 3 seconds auto-dismiss
- Colour: success green `#22C55E`
- Text: "Video published successfully" (on publish) | "Notification sent" (on send)
- Applies to: AdminPage after publish + after send notification

### Real-Time Status (Task A-8)
Source: story-6.md User Flow (Send Notification panel)

- Status updates in real-time via `onSnapshot` listener
- Listener must be cleaned up on component unmount (use ref pattern)
- UI transitions: pending → spinner/text → sent (green) or failed (red)

### Thumbnail Auto-Populate (Task A-7)
Source: story-6.md User Flow step 3

- On blur of youtubeVideoId field
- Auto-populate: `https://img.youtube.com/vi/{videoId}/hqdefault.jpg`
- Show inline thumbnail preview image at same time
- Editable after auto-populate (admin can override with custom URL)
