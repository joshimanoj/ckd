## 1. Relevant Components

### Sign In Screen (OnboardingPage)

**States:**
- **Idle:** creator photo + app name + "Continue with Google" button. Fully interactive.
- **Loading:** button disabled, label "Signing in…", spinner replaces Google icon. aria-busy="true".
- **Error (network):** button re-enabled; inline error below button in `#EF4444`, 13sp Nunito: "Sign in failed. Please try again."
- **Error (cancel):** no error shown; button re-enabled silently.
- **Offline:** entire sign-in UI replaced by offline screen — "Check your connection" text + Retry button. No creator photo or button visible.

**Layout:**
- Upper half: brand gradient `linear-gradient(135deg, #F43F5E 0%, #9333EA 50%, #EC4899 100%)` — 50vh
- Creator photo: 120px diameter, circular, gold ring border (`#F59E0B`, 3px), vertically centred straddling the gradient/surface boundary
- Lower half: `#FAFAFA` surface
- App name: centred below photo, `#9333EA`, 28sp Baloo 2 ExtraBold 800
- Button container: max-width 360px, centred, 32dp horizontal padding
- "Continue with Google" button: `#F43F5E`, 48px height, 24px border-radius (pill), full container width, Nunito SemiBold 600, white label, Google "G" SVG icon left of label

**Animations:**
- Button press: scale 0.95 on active state, 100ms ease-out (standard tap feedback — Design.md §7)
- Offline screen: fade in (opacity 0 → 1, 200ms)

**Layout rules:**
- No horizontal scroll at any viewport ≥ 320px
- Responsive at 375px, 480px, 768px — all elements centre-aligned, no overflow
- No top navigation bar on the Sign In screen (auth stack is outside the main nav shell)
- Touch target minimum: 48×48px on button

## 2. Relevant User Flows

### Flow 1: First-Time Onboarding (Sign In step)

Source: Design.md §4, Flow 1 — steps 1 only (Story #1 scope ends at redirect to Consent)

1. **Sign In screen** → parent taps "Continue with Google" → Google sign-in popup opens → parent selects Google account → returns to app
2. → Firestore user doc created with `consentGiven: false`
3. → App detects `consentGiven: false` → routes to `/consent` (Story #2 handles from here)

**Success state:** Sign In screen transitions away; no lingering on the Sign In route.
**Error state — network failure:** Inline error below button: "Sign in failed. Please try again." Retry available. No crash, no blank screen.
**Error state — cancel:** No error message. Sign In screen remains, button re-enabled.
**Error state — offline on launch:** Offline screen shown before Sign In screen. Retry restores Sign In screen when connection returns.

### Return Visit Flow

1. App loads → `onAuthStateChanged` fires → user already authenticated
2. `useAuth` reads `consentGiven` from Firestore:
   - `false` → route to `/consent`
   - `true` + child profile exists → route to `/library`
   - `true` + no child profile → route to `/profile`
3. Sign In screen never shown to authenticated user

### Sign-Out Flow

1. `signOut(auth)` called → `onAuthStateChanged` fires → user is null → route to `/`
2. Sign In screen shown

## 4. Interaction Rules

Source: Design.md §7 — rules applicable to components in this story only.

| Rule | Detail | Applies to |
|---|---|---|
| Button tap feedback | Scale to 0.95, 100ms ease-out on `:active` | GoogleSignInButton |
| Error display | Inline below the triggering element, 13sp Nunito, `#EF4444`, shown on blur or after failed action | GoogleSignInButton error state |
| Loading state | Disable interactive element, show spinner in place of icon, update label to "Signing in…" | GoogleSignInButton loading state |
| Error state — cancel | `auth/popup-closed-by-user` is a user intent signal, not an error — show nothing | GoogleSignInButton |
| Offline state | Full-screen replacement of content area (not a modal) with "Check your connection" + Retry | OnboardingPage offline state |
| Retry button | Re-checks `navigator.onLine`, restores Sign In screen if online | OnboardingPage offline Retry |
| Touch target | Minimum 48×48px on all interactive elements | GoogleSignInButton, Retry button |
| No navigation bar | Auth stack screens have no top or bottom navigation bar | OnboardingPage |
