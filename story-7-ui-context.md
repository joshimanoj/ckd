## 1. Relevant Components

### PlayerScreen
Full-screen container wrapping the YouTube iframe and controls overlay.

- Background: `#000000` (black) while video loads and during playback
- Layout: 100vw × 100vh, `position: fixed`, `top: 0`, `left: 0`, `z-index: 50`
- No bottom navigation bar — hidden on this screen
- No category chips, no video title bar — fully immersive
- Top navigation bar rendered on web (replaces mobile bottom nav) — per Design.md §9
- `overflow: hidden` — prevent any horizontal scroll
- Web: `position: fixed` full-screen iframe, `width: 100%`, `height: 100%`, border: none

### PlayerControls
Overlay rendered on top of the iframe. Two zones:

**Top bar (back button):**
- Position: `position: absolute`, `top: 0`, `left: 0`, `right: 0`, height: 56dp
- Background: `rgba(0, 0, 0, 0.4)` (semi-transparent dark scrim)
- Back button: white left-arrow icon, `min-width: 44px`, `min-height: 44px` tap area, top-left

**Bottom bar (play/pause):**
- Position: `position: absolute`, `bottom: 0`, `left: 0`, `right: 0`, height: 120dp
- Background: `rgba(0, 0, 0, 0.4)` (semi-transparent dark scrim)
- Play/Pause button: white icon (32dp), inside circular button `width: 56px`, `height: 56px`, `border-radius: 50%`, background: `rgba(0, 0, 0, 0.6)`, centered in bottom bar
- Button text/aria-label: "Play" or "Pause" depending on state (screen reader support)

States:
- `idle` — controls overlay hidden after 3s inactivity (tap screen to reveal)
- `active` — controls overlay visible
- `loading` — black screen + centered white spinner (ActivityIndicator equivalent: CSS spinner or `<LoadingSpinner />`)
- `error` — error overlay replaces player area

### Player Error State
- Container: `position: absolute`, `inset: 0`, `background: #000000`, `display: flex`, `flex-direction: column`, `align-items: center`, `justify-content: center`
- Text: "Oops! Check your internet connection." — Nunito Regular 16sp, color: `#FFFFFF`
- Button: "Try again" — outlined white, pill (`border-radius: 24px`), white border + white label
- No illustration in V1 web (illustration asset not available — use text + icon)

### Player Loading State
- Container: full-screen black (`#000000`)
- Spinner: centered, white, standard CSS animation or `<LoadingSpinner />` component

## 2. Relevant User Flows

### Flow 2: Child Watches a Video (Design.md §4)

1. Video Library → tap thumbnail → Player screen pushes in (`/watch/:videoId`)
2. Player screen → video loads and auto-plays full-screen → custom play/pause visible → watch session begins in memory
3. Video ends → session written to Firestore → auto-advance to next video (loops to first if at end of library)
4. Child/parent taps back button → player dismissed → returns to `/library` → session written to Firestore

**Success state:** Video plays without interruption, no ads, no external links.
**Error state:** Video fails to load → "Oops! Check your internet connection." + retry button.

## 5. Web-Specific Rules (Design.md §9)

- Bottom navigation bar → replaced by **top navigation bar** on web
- Player screen on web: top nav still renders (unlike mobile which has no nav bar on player)
- Primary mobile web target: **480–768px** viewport width
- No horizontal scroll at `< 480px`
- Iframe: `width: 100%`, `height: 100%`, `border: none`, `display: block`
- Player container: `position: fixed`, `inset: 0` — fills entire viewport including nav area
- On web, the top nav renders as a transparent overlay at top of player (not a separate layout element that reduces player height)
