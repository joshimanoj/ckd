# UAT Checklist: Story #10 — Settings Screen & Privacy Policy Link

Generated by /prd | Sources: story-10.md (acceptance criteria, user flows), story-10-ui-context.md

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
**Test account:** Sign in with a Google account that has previously completed consent + child profile setup. If starting fresh, complete Stories 1–3 first.
**Start at:** Video Library screen (`/library`)

---

## Checklist

### Flow 1: Access Settings via Parent Panel

- [ ] 1. From Video Library, tap the 🔒 parent icon in the top-right header
- [ ] 2. Verify: Parental Gate modal appears with a maths question (e.g. "What is 3 + 8?")
- [ ] 3. Enter the correct answer and tap "Submit"
- [ ] 4. Verify: Parent Panel drawer slides in from the right
- [ ] 5. Verify: Two tabs are visible — "📊 Dashboard" and "⚙️ Settings"
- [ ] 6. Tap "⚙️ Settings"
- [ ] 7. Verify: Settings screen renders (notification toggle, Privacy Policy, Sign Out, version footer all visible)

### Flow 2: Notification Toggle

- [ ] 8. Verify: toggle label reads "New video notifications" (not "New video alerts")
- [ ] 9. Verify: toggle subtitle reads "Get notified when new rhymes are added"
- [ ] 10. Verify: toggle reflects the current saved preference (OFF by default for a new account)
- [ ] 11. Tap the toggle
- [ ] 12. Verify: Parental Gate appears again (gate is required for toggle changes)
- [ ] 13. Enter the correct answer
- [ ] 14. Verify: toggle flips to ON (or OFF if it was ON)
- [ ] 15. Verify: success toast appears briefly at the bottom ("Notifications turned on" or "turned off")
- [ ] 16. Close and reopen the panel — verify the toggle still reflects the new saved state

### Flow 3: Privacy Policy Link

- [ ] 17. In Settings, locate the "Privacy Policy ›" link
- [ ] 18. Verify: link text colour is purple (`#9333EA`), underlined, with a `›` chevron
- [ ] 19. Verify: link is NOT greyed out or disabled
- [ ] 20. Tap/click the Privacy Policy link
- [ ] 21. Verify: a new browser tab opens (not an in-app navigation)
- [ ] 22. Verify: Settings panel remains open (no in-app navigation occurred)

### Flow 4: Sign Out — Cancel

- [ ] 23. In Settings, locate the "Sign Out" row
- [ ] 24. Verify: "Sign Out" label is red (`#EF4444`)
- [ ] 25. Tap "Sign Out"
- [ ] 26. Verify: confirmation dialog appears: "Are you sure you want to sign out?"
- [ ] 27. Verify: dialog has "Sign Out" (red) button and "Cancel" button
- [ ] 28. Tap "Cancel"
- [ ] 29. Verify: dialog closes, Settings screen is still visible
- [ ] 30. Verify: user is still on `/library` (not signed out)

### Flow 5: Sign Out — Confirm

- [ ] 31. In Settings, tap "Sign Out" again
- [ ] 32. Tap "Sign Out" (red confirm button) in the dialog
- [ ] 33. Verify: user is redirected to the sign-in screen (`/`)
- [ ] 34. Verify: the sign-in screen shows the Google Sign-In button (user is fully signed out)
- [ ] 35. Verify: navigating directly to `/library` in the browser redirects back to `/` (stores cleared, not just sign-in redirect)

### Flow 6: App Version Footer

- [ ] 36. Re-sign in and navigate back to Settings
- [ ] 37. Scroll to the bottom of the Settings panel
- [ ] 38. Verify: "Version 1.0.0" (or current version) text appears, centred, grey
- [ ] 39. Verify: version text is NOT tappable (no cursor-pointer, no navigation)

### Visual Check

- [ ] 40. All text in Settings uses Nunito font (not Baloo 2, which is used for child-facing headings)
- [ ] 41. "ACCOUNT" section header visible above Sign Out, in uppercase grey small text
- [ ] 42. A thin horizontal divider (`#E5E7EB`) separates the notification section from the Account section
- [ ] 43. Toggle rows have a white card background with subtle shadow (not flat on grey)
- [ ] 44. "Sign Out" label is clearly red — distinct from other rows
- [ ] 45. No gamified elements, illustrations, or emoji (except the 🔒 lock note on the toggle)

### Responsive / Device Check

- [ ] 46. Open DevTools → set viewport to 480px wide — verify Settings renders without horizontal scroll
- [ ] 47. Set viewport to 375px wide — verify no content is cut off or overflows
- [ ] 48. Set viewport to 768px wide — verify all rows are visible and correctly spaced
- [ ] 49. Verify Parent Panel drawer width stays within viewport at all widths (max 320px)

### Edge Cases

- [ ] 50. Notification toggle OFF while offline (disconnect network) → toggle tap → after gate → verify error toast appears and toggle reverts (if Firestore write fails)
- [ ] 51. Privacy Policy URL not configured (`VITE_PRIVACY_POLICY_URL` env var not set) → link still renders (falls back to `href="#"`) — no broken layout
