# UAT Checklist: Story #2 — DPDP Parental Consent Flow

Generated by /prd | Sources: story-2.md (acceptance criteria, user flows), story-2-ui-context.md

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
**Emulator:** Firebase emulator must be running: `firebase emulators:start`
**Start at:** Sign In screen (unauthenticated, fresh emulator state)
**Test account:** Create a fresh test user via emulator (or clear emulator data before each run)

---

## Checklist

### Flow 1: Happy Path — First-Time Consent

- [ ] 1. Open `http://localhost:5173` — Sign In screen visible, not redirected
- [ ] 2. Sign in with test Google account
- [ ] 3. Consent modal appears automatically — title "Before we begin" visible
- [ ] 4. Modal is over the full screen — no other content accessible beneath it
- [ ] 5. Data list is visible and contains exactly 4 bullet points:
  - "Your name and email (from your Google account)"
  - "Your child's name and date of birth"
  - "How long your child watches each video (watch time data)"
  - "Your device token (to send you notifications, only if you opt in)"
- [ ] 6. "Why it's collected" paragraph is visible with correct text
- [ ] 7. Checkbox is **unchecked** on load (no pre-check)
- [ ] 8. "I Agree & Continue" button is **disabled** (visually grey, not clickable)
- [ ] 9. Tap "Privacy Policy" link — opens in a new browser tab (not in-app)
- [ ] 10. Check the checkbox — button turns coral-red and becomes clickable
- [ ] 11. Tap "I Agree & Continue" — navigates to `/profile` (Child Profile screen)
- [ ] 12. Consent modal is no longer visible after navigation

### Flow 2: Non-Dismissable Behaviour

- [ ] 13. On `/consent`, click the dark overlay area outside the modal card — modal stays open, URL stays `/consent`
- [ ] 14. Press `Escape` key — modal stays open
- [ ] 15. Press browser back button — app redirects back to `/consent` within 2 seconds

### Flow 3: Already Consented User

- [ ] 16. Sign out, then sign in as a user who previously completed consent
- [ ] 17. Consent modal does **not** appear — user is routed directly to `/profile` or `/library`

### Flow 4: Error State

- [ ] 18. With browser DevTools → Network tab, block requests to `127.0.0.1:8080` (Firestore emulator)
- [ ] 19. On `/consent`, check checkbox and click "I Agree & Continue"
- [ ] 20. Error toast appears: "Something went wrong. Please try again."
- [ ] 21. Modal remains open — no navigation to `/profile`
- [ ] 22. Restore network — retry works, navigates to `/profile`

### Visual Check

- [ ] 23. Title "Before we begin" uses rounded/bold font (Baloo 2), colour `#1E1B4B` (deep indigo)
- [ ] 24. Body text is readable at 15sp, line-height comfortable (1.5)
- [ ] 25. Checkbox label includes "Privacy Policy" as an underlined purple link
- [ ] 26. Checkbox hit area feels generous (≥48×48dp) — easy to tap
- [ ] 27. Modal card has noticeably rounded corners (20dp radius)
- [ ] 28. "I Agree & Continue" button height is at least 48dp, pill-shaped (24dp radius)
- [ ] 29. No creator photo in this modal (legal screen — correct by design)
- [ ] 30. No external links other than Privacy Policy

### Responsive / Device Check

- [ ] 31. At 480px viewport width (DevTools → Responsive) — modal card fits, no horizontal scroll
- [ ] 32. At 375px viewport width — still no horizontal scroll, text readable
- [ ] 33. At 768px viewport width — modal centred, max-width respected (not full-width)
- [ ] 34. On mobile web (real phone browser) — modal fills screen appropriately, checkbox is tappable

### Edge Cases

- [ ] 35. Kill and reopen the browser tab while on `/consent` before confirming — modal shows again with checkbox unchecked
- [ ] 36. Check checkbox, then uncheck — button becomes disabled again (grey) without page reload
- [ ] 37. Double-tap "I Agree & Continue" rapidly — only one Firestore write occurs (button disabled during submitting)
