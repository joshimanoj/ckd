# UAT Checklist: Story #9 — Push Notifications

Generated by /prd | Sources: story-9.md (acceptance criteria, user flows), story-9-ui-context.md (components, tokens)

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
**Test data:**
- Sign in with your Google account (used for testing)
- Ensure at least one video exists in the Firestore library
- Use a browser that supports push notifications (Chrome recommended)
- Clear localStorage before starting: DevTools → Application → Storage → Clear site data

**Start at:** Sign In screen (or Library if already signed in)

**Pre-check:** Open DevTools → Application → Service Workers and confirm `firebase-messaging-sw.js` is registered at scope `/`

---

## Checklist

### Flow 1: Opt-in prompt appears after first video

[ ] 1. Sign in and navigate to the Video Library
[ ] 2. Tap any video thumbnail — player opens and video begins
[ ] 3. Wait for the video to end (or use browser DevTools console: `window.__simulateVideoEnd()`)
[ ] 4. Verify: an in-app bottom sheet slides up from the bottom of the screen
[ ] 5. Verify: sheet shows a bell icon (🔔) in brand purple
[ ] 6. Verify: title reads "Get notified about new rhymes!" in dark indigo bold text
[ ] 7. Verify: subtitle reads "We'll let you know when new videos are added." in grey text
[ ] 8. Verify: "Yes, notify me" button is coral-red, full-width, at least 48px tall
[ ] 9. Verify: "Not now" appears below — ghost style, violet text, at least 48px tall

### Flow 2: "Not now" dismisses sheet — no notification granted

[ ] 1. With opt-in sheet visible, tap "Not now"
[ ] 2. Verify: sheet slides down and disappears
[ ] 3. Verify: browser does NOT show a notification permission dialog
[ ] 4. Verify: player navigates to the next video as normal
[ ] 5. Watch a second video to the end — verify opt-in sheet does NOT appear again
[ ] 6. Open Firestore console (or DevTools Network) — verify `notificationsEnabled` is still `false`

### Flow 3: "Yes, notify me" → permission granted → token written

[ ] 1. Clear localStorage (DevTools → Application → Storage → Clear site data) to reset the "prompted" flag
[ ] 2. Watch a video to the end — opt-in sheet appears
[ ] 3. Tap "Yes, notify me"
[ ] 4. Verify: browser notification permission dialog appears (Chrome popup)
[ ] 5. Click "Allow" in the browser dialog
[ ] 6. Verify: sheet dismisses
[ ] 7. Open Firebase Console (Firestore) → `users/{your-uid}` → verify:
     - `notificationsEnabled: true`
     - `fcmToken: <non-null string>`
[ ] 8. Verify: player continues to next video normally after sheet dismisses

### Flow 4: Token refreshed on app reload

[ ] 1. After opting in (notificationsEnabled: true), reload the page
[ ] 2. Open Firebase Console → `users/{your-uid}` → verify `fcmToken` is still populated (not null)
[ ] 3. Note the token value — it should be the same or a freshly rotated token (both are correct)

### Flow 5: Settings tab — notification toggle with Parental Gate

[ ] 1. On the Library screen, tap the parent icon (top-right)
[ ] 2. Verify: Parent Panel drawer opens from the right
[ ] 3. Tap the "Settings" tab
[ ] 4. Verify: Settings screen shows a "Notifications" section heading
[ ] 5. Verify: "New video alerts" toggle row is visible with a lock icon note "Requires parent verification"
[ ] 6. Tap the notification toggle
[ ] 7. Verify: Parental Gate modal appears (math question)
[ ] 8. Enter an incorrect answer — verify toggle does NOT change
[ ] 9. Tap the toggle again → gate appears again → enter correct answer
[ ] 10. Verify: toggle changes state (on → off, or off → on)
[ ] 11. Open Firebase Console → `users/{your-uid}` → verify `notificationsEnabled` matches toggle state
[ ] 12. Verify: a success toast (green, "Saved" or similar) appears briefly at the bottom

### Flow 6: Push notification received (admin sends)

[ ] 1. Open the Admin Panel (`/admin`)
[ ] 2. Click "Send Notification" in the Notification Panel section (Story 6)
[ ] 3. Enter title: "Test Notification" and body: "New rhyme is here!"
[ ] 4. Click "Send"
[ ] 5. Verify: notification document status updates to "sent" in Firestore (Admin Panel or Firestore Console)
[ ] 6. On the same device/browser (Chrome): verify a browser push notification appears at OS level
[ ] 7. Click the notification — verify: app opens to `/library`
[ ] 8. Verify: the most recently published video appears at the top of the grid

### Flow 7: Permission denied — graceful handling

[ ] 1. Clear localStorage, revoke notification permission in Chrome Settings for localhost
[ ] 2. Watch a video to end — opt-in sheet appears
[ ] 3. Tap "Yes, notify me"
[ ] 4. Verify: Chrome permission dialog appears → click "Block"
[ ] 5. Verify: sheet dismisses gracefully (no crash, no error screen)
[ ] 6. Verify: app continues to next video
[ ] 7. Verify: `notificationsEnabled` remains `false` in Firestore

## Visual Check

[ ] Bell icon in opt-in sheet is brand purple `#9333EA`
[ ] "Yes, notify me" button is coral-red `#F43F5E` — matches brand accent colour
[ ] "Not now" button has no fill — ghost style with violet label
[ ] Sheet background is `#FAFAFA` (near-white, clean)
[ ] Title uses Baloo 2 Bold — rounded, friendly letterforms
[ ] Subtitle uses Nunito Regular — legible, smaller text
[ ] Toggle in Settings uses `#9333EA` when active
[ ] Success toast is green, slides up from bottom, disappears after ~3 seconds
[ ] Parental Gate modal appears with correct styling (rounded, centred)

## Edge Cases

[ ] Watch multiple videos in one session — opt-in sheet only appears once (after first video end)
[ ] Already opted in: watch a video to end — opt-in sheet does NOT appear
[ ] Reload page after opting out via Settings toggle — `notificationsEnabled: false` persists
[ ] Sign out and sign back in — FCM token is still written (or refreshed) on next sign-in if permission is granted
