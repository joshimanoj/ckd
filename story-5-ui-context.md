## 1. Relevant Components

### App Header
**States:** Static (no interactive states — parent icon is the only interactive element)
**Layout rules:**
- Height: 64px (web — slightly taller than mobile 56px for visual weight)
- Background: `linear-gradient(135deg, #F43F5E 0%, #9333EA 50%, #EC4899 100%)`
- Layout: row, `align-items: center`, `padding: 0 16px`
- Creator avatar: 40dp circular, left-aligned, `margin-right: 12px`, `object-fit: cover`, gold/gradient ring border (optional — simplified to circular crop in V1)
- App title: "Choti Ki Duniya", Baloo 2 Bold 18sp, white, flex-grow 1
- Parent icon: top-right of header, 32dp visual, 44×44dp tap target, white lock emoji or icon, transparent background

### Parent Icon Button
**States:** Idle → triggers Parental Gate on tap (Story #4)
**Layout rules:**
- Position: absolute right 8px OR flex end in header
- Tap target: 44×44dp (min — per Design.md §8)
- Visual: white circular lock, slight shadow
- The gate modal (ParentalGate component) already exists from Story #4 — do not rebuild

### VideoCard
**States:**
- Idle: card at rest, thumbnail loaded
- Active (pressed): `scale(0.95)` transform, 100ms transition
- Thumbnail error: grey `#E5E7EB` placeholder fills 16:9 area, no broken icon
**Layout rules:**
- Card background: `#FAFAFA`, `border-radius: 16px`, `box-shadow: 0 4px 4px rgba(147,51,234,0.12)`
- Thumbnail: 16:9 aspect ratio (padding-top: 56.25%), `border-radius: 16px`, `overflow: hidden`, `loading="lazy"`
- Category chip: overlaid bottom-left of thumbnail, `position: absolute`, `bottom: 8px`, `left: 8px`
- Title: below thumbnail, `padding: 8px`, Baloo 2 SemiBold 15sp, `color: #1E1B4B`, 2-line clamp
- No external links, no YouTube logo, no "Subscribe" CTA

### Category Chip (on VideoCard)
- `background: #F3E8FF`, `color: #9333EA`, `border-radius: 20px`, `padding: 4px 10px`
- Font: Nunito SemiBold 13sp

### CategoryFilter
**States:**
- Hidden: `display: none` when `videoCount < 20`
- Visible: horizontal scrollable row when `videoCount >= 20`
- Chip unselected: `background: #F3E8FF`, `color: #9333EA`
- Chip selected: `background: #9333EA`, `color: white`
**Layout rules:**
- `display: flex`, `overflow-x: auto`, `gap: 8px`, `padding: 8px 16px`
- No visible scrollbar (`::-webkit-scrollbar { display: none }` or `scrollbar-width: none`)
- Chips: `border-radius: 20px`, `padding: 6px 16px`, Nunito SemiBold 13sp, `border: none`, `cursor: pointer`
- "All" chip is always first

### VideoGrid (grid container)
**Layout rules:**
- `display: grid`, `grid-template-columns: repeat(2, 1fr)`, `gap: 8px`, `padding: 16px`
- Viewport `< 480px`: maintain 2 columns (do not collapse to 1 — story spec requires no horizontal scroll at < 480px while maintaining 2 cols)

### SkeletonGrid
**States:** One state — animated shimmer
**Animation:**
```css
@keyframes shimmer {
  0%   { background-position: 200% 0 }
  100% { background-position: -200% 0 }
}
```
- `background: linear-gradient(90deg, #F3E8FF 25%, #E9D5FF 50%, #F3E8FF 75%)`
- `background-size: 200% 100%`
- `animation: shimmer 1.4s infinite linear`
- Same grid layout as VideoGrid (2 columns, 8px gap, 16px padding)
- Each skeleton card has the same 16:9 aspect ratio wrapper as VideoCard

### EmptyState (0 active videos)
- Centered vertically and horizontally in the grid area
- `data-testid="empty-state"`
- Heading: "Videos coming soon!" — Baloo 2 Bold 20sp, `color: #1E1B4B`
- Subtitle: "Check back soon for new rhymes." — Nunito Regular 15sp, `color: #6B7280`
- Illustration: creator photo or placeholder image (use `/src/assets/creator-photo.jpg`)
- No external links

### ErrorState (offline / fetch failure)
- Full-width within grid area (not a modal)
- `data-testid="error-state"`
- Message: "Check your internet connection" — Nunito Regular 15sp
- Retry button: `data-testid="retry-btn"`, primary style (filled `#7C3AED`, white label, 48px height min)

## 2. Relevant User Flows

### Flow 2 — Child Watches a Video (Step 1)
**Source:** Design.md §4, Flow 2

Steps this story touches:
1. Video Library screen mounts → `useVideoLibrary` hook fetches active videos
2. Grid renders with video cards
3. Child or parent taps thumbnail → navigate to Video Player (`/watch/:videoId`) passing `videoId`

Success state: Grid visible, videos loaded, tap navigation works
Error state: No internet → `data-testid="error-state"` full-screen, retry button
