# UAT Checklist: Story #5 — Video Library Grid

Generated by /prd | Sources: story-5.md (acceptance criteria, user flows), story-5-ui-context.md

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
**Firebase:** Requires Firebase emulator OR real Firestore with at least 2–3 seed videos
**Test data:**
- Sign-in account: any Google account that has completed onboarding (consent + child profile)
- Seed videos via Admin Panel (Story #6) OR directly in Firestore emulator
- For category filter test: need 20+ active videos

**Start at:** Sign In screen (or directly at `/library` if already authenticated)

---

## Checklist

### Flow 1: Library loads and displays videos

- [ ] 1. Sign in and complete onboarding → arrive at Video Library screen
- [ ] 2. While videos are loading: shimmer skeleton grid is visible (same 2-column shape as cards)
- [ ] 3. After videos load: skeleton disappears, video grid appears
- [ ] 4. Grid renders in 2 columns
- [ ] 5. Each card shows: thumbnail image (16:9 ratio), video title below, category chip overlaid bottom-left of thumbnail
- [ ] 6. App header shows: creator avatar (circular, 40px) on the left, "Choti Ki Duniya" title in white, lock icon on the right
- [ ] 7. Header background is the brand gradient (red → purple → pink)
- [ ] 8. Screen background is light purple (#F3E8FF)

### Flow 2: Video card interaction

- [ ] 9. Tap a video card — it briefly scales down (tactile feedback) then navigates to the video player screen
- [ ] 10. URL changes to `/watch/{videoId}` after tapping

### Flow 3: Pull-to-refresh

- [ ] 11. Scroll down, then scroll back to the top (or tap the refresh button)
- [ ] 12. Library re-fetches — skeleton appears briefly, then updated video list appears

### Flow 4: Parent icon and Parental Gate

- [ ] 13. Tap the lock icon (top-right of header)
- [ ] 14. Parental Gate math question modal appears
- [ ] 15. Answer correctly → Parent Panel appears (stub: "Parent Panel (Story 8)")
- [ ] 16. Answer wrongly → input clears, new question appears, modal stays open
- [ ] 17. Tap X / dismiss → gate closes, library visible, no parent panel

### Flow 5: Empty state

- [ ] 18. With zero active videos in Firestore: empty state shows "Videos coming soon!" with creator image
- [ ] 19. Subtitle "Check back soon for new rhymes." visible
- [ ] 20. No video grid visible when empty

### Flow 6: Offline / error state

- [ ] 21. Disable network (DevTools → Network → Offline) and navigate to library
- [ ] 22. Error state shows: message "Check your internet connection"
- [ ] 23. "Try again" button visible
- [ ] 24. Re-enable network, tap "Try again" → library loads successfully

### Flow 7: Category filter (requires 20+ active videos)

- [ ] 25. With < 20 videos: category filter row is NOT visible
- [ ] 26. With 20+ videos: category filter row IS visible below the header
- [ ] 27. Filter chips include "All" plus each category present in the video list
- [ ] 28. Tapping "Rhymes" chip → grid filters to show only Rhymes videos
- [ ] 29. Selected chip has purple background, white text
- [ ] 30. Tapping "All" chip → all videos shown again

---

## Visual Check

- [ ] Card thumbnails are 16:9 ratio (wider than tall, not square)
- [ ] Card corners are rounded (16px radius)
- [ ] Card has subtle purple-tinted shadow
- [ ] Category chip on card uses light purple background + purple text, pill shape
- [ ] Card title is 2 lines max, truncated with ellipsis if longer
- [ ] Skeleton shimmer animates (not static — gradient sweeps across)
- [ ] Header gradient is clearly visible (red-to-purple-to-pink, not solid colour)
- [ ] Creator avatar is circular with no distortion

---

## Responsive / Device Check

- [ ] At 480px viewport width: 2-column grid renders correctly, no horizontal scroll
- [ ] At 375px viewport width: 2-column grid renders, no horizontal scroll, all content fits
- [ ] At 768px viewport width: 2-column grid renders, correct spacing
- [ ] At any width: no content is cut off or overflows the screen horizontally

---

## Edge Cases

- [ ] Broken thumbnail URL → grey placeholder fills the 16:9 area, no broken image icon, title and chip still visible
- [ ] Very long video title (30+ characters) → truncated to 2 lines with "…", card height does not grow abnormally
- [ ] Returning to library after watching a video (pressing back from `/watch/:videoId`) → grid loads immediately from cache, no re-fetch
- [ ] Pull-to-refresh while offline → error toast or retry state, no crash
