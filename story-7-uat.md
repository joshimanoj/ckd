# UAT Checklist: Story #7 — Video Player & Watch Session Tracking

Generated by /prd | Sources: story-7.md (acceptance criteria, user flows), story-7-ui-context.md (components, tokens)

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
**Test data:** Sign in with your Google account → ensure at least one child profile exists → seed 3+ videos via Admin Panel (`/admin`)
**Start at:** Video Library screen (`/library`) — logged in, child profile set up, videos visible in grid

---

## Checklist

### Flow 1: Tap a video and watch it

- [ ] 1. Tap any video thumbnail in the Library grid
- [ ] 2. Verify the player screen loads — full-screen black background, no Library visible behind it
- [ ] 3. Verify the YouTube video begins playing automatically (no manual tap needed)
- [ ] 4. Verify there are NO pre-roll ads before playback
- [ ] 5. Verify there is NO visible YouTube player toolbar (no progress bar, volume controls, fullscreen button from YouTube)
- [ ] 6. Verify the custom play/pause button is visible at the bottom of the screen — circular, white icon on dark translucent background
- [ ] 7. Verify the back arrow button is visible at the top-left — white icon on dark translucent strip
- [ ] 8. Verify no bottom navigation bar is visible on the player screen
- [ ] 9. Verify no video title, category chips, or other Library UI elements appear on the player screen

### Flow 2: Play/Pause

- [ ] 1. While a video is playing, tap the play/pause button
- [ ] 2. Verify the video pauses
- [ ] 3. Verify the button icon changes (from pause ⏸ to play ▶)
- [ ] 4. Tap again — verify the video resumes playing
- [ ] 5. Verify the button tap area is comfortably large (at least 56px × 56px — easy to tap)

### Flow 3: Back to Library

- [ ] 1. While a video is playing, tap the back button (top-left)
- [ ] 2. Verify the player dismisses
- [ ] 3. Verify you are returned to the Video Library (`/library`)
- [ ] 4. Verify the Library scroll position is preserved (you see the same videos as before)

### Flow 4: Video ends — auto-advance

- [ ] 1. Let a short video play to the end (or seek via browser DevTools if needed)
- [ ] 2. Verify the next video in the library begins playing automatically
- [ ] 3. Verify the URL changes to `/watch/<next-video-id>`
- [ ] 4. Let the last video in the library end
- [ ] 5. Verify playback wraps around to the first video in the library

### Flow 5: Watch session tracking

- [ ] 1. Watch a video for at least 15–20 seconds, then tap back
- [ ] 2. Open Firebase Console (or Firestore emulator) and check: `users/{uid}/childProfiles/{childProfileId}/watchSessions`
- [ ] 3. Verify a new document was written with:
  - `youtubeVideoId` matching the video you watched
  - `watchedSeconds` ≈ 15–20 (approximately, not exact — depends on polling)
  - `completionPercent` a non-zero integer
  - `startTime` a valid timestamp
  - `endTime` a valid timestamp (not null)
  - `deviceType: 'web'`
- [ ] 4. Watch the same video again and pause mid-way — verify a second document written (not an update to the first)
- [ ] 5. Verify no more than 1 Firestore document per session (no duplicate writes from rapid back/pause taps)

### Flow 6: Error state

- [ ] 1. Disconnect from Wi-Fi / set browser to Offline in DevTools Network tab
- [ ] 2. Tap a video thumbnail from the Library
- [ ] 3. Verify an error message appears inside the player: "Oops! Check your internet connection."
- [ ] 4. Verify a "Try again" button is visible
- [ ] 5. Reconnect to the network
- [ ] 6. Tap "Try again" — verify the player reinitializes and attempts to load the video

---

## Visual Check

- [ ] Player background is pure black (`#000000`) while loading and during playback
- [ ] Custom play/pause button: white icon (≈32px) inside circular button (56px), dark translucent background
- [ ] Back button: white left-arrow, top-left corner, dark translucent top bar
- [ ] Error text: white, Nunito font, centered on black background
- [ ] "Try again" button: outlined white, pill shape — NOT a filled purple button
- [ ] No YouTube logo or watermark prominent during playback (modestbranding applied)
- [ ] Controls overlay does not obscure the centre of the video (content is in the safe zone)

---

## Responsive / Device Check

- [ ] **375px mobile width** — open DevTools, set viewport to 375×812px; verify player fills screen, no horizontal scrollbar, back and play/pause buttons visible
- [ ] **480px mobile width** — set viewport to 480×844px; verify player fills width, controls visible, no overflow
- [ ] **768px tablet width** — set viewport to 768×1024px; verify player fills full width, no empty side gaps

---

## Edge Cases

- [ ] Navigate directly to `/watch/invalid-video-id` — verify either the error state is shown or you are redirected to `/library`
- [ ] Open two browser tabs — both on different watch pages — verify each session tracks independently (two separate documents in Firestore)
- [ ] If a video has `durationSeconds: 0` in Firestore — verify the player loads without crashing and `completionPercent: 0` in the written session document
