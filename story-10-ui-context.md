## 2. Relevant User Flows

### Flow: Parent Panel → Settings tab (existing, from Design.md §3)
- Video Library → parent icon → Parental Gate → Parent Panel opens (side drawer on web)
- Tab bar: "📊 Dashboard" | "⚙️ Settings" — Nunito SemiBold 13sp
  - Active: `#7C3AED` with white background card; Inactive: `#9CA3AF` on `#F3E8FF` background
- Settings tab renders SettingsScreen

### Flow: Privacy Policy link
1. Settings tab active
2. Parent taps "Privacy Policy ›" link
3. Browser opens `VITE_PRIVACY_POLICY_URL` in a new tab (`target="_blank"`)
4. Settings remains open (no navigation within app)

**Success state:** New tab opens to privacy policy page.
**Error state (URL not set):** Link renders as `href="#"` — does nothing. This is acceptable during development; must be set before Play Store submission.

### Flow: Sign Out
1. Settings tab active
2. Parent taps "Sign Out" row
3. Confirmation dialog overlays everything
4a. Parent taps "Sign Out" (red): `onSignOut()` called → stores cleared → Firebase signOut → `onAuthStateChanged` fires null → AuthGuard redirects to `/` (sign-in screen)
4b. Parent taps "Cancel": dialog dismissed, settings stays open

**Success state:** User lands on sign-in screen (`/`), all stores cleared.
**Error state:** `signOut()` throw → dialog closes but user may still be on library — acceptable (Firebase signOut is rarely failure; no error handling required for V1).

## 4. Interaction Rules

| Rule | Description | Applies to |
|---|---|---|
| Parental Gate required | Toggle changes require gate before state changes | Notification Toggle Row |
| Destructive confirmation | Any destructive action requires a confirmation dialog (not browser native confirm) | Sign Out Row |
| External link | Opens `target="_blank"` — never in-app WebView on parent screens | Privacy Policy Row |
| Touch target minimum | 48dp min height/width on all interactive elements | All rows, all buttons |
| Responsive overflow | `overflowX: 'hidden'`, `boxSizing: 'border-box'`, `maxWidth: '100%'` on container | SettingsScreen container |
| Settings is adult-tone | Nunito (not Baloo 2) for all body/label text; no gamified elements | Entire SettingsScreen |
| Sign Out is destructive | Red label (`#EF4444`), confirmation required, aligns with Design.md §7 destructive pattern | Sign Out button + dialog |
