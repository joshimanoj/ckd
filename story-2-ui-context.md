## 1. Relevant Components

### ConsentModal
Full-screen non-dismissable overlay modal. Renders over all other content. No creator photo — this is a legal/trust screen.

**States:**
- **Idle** — modal open, checkbox unchecked, button disabled (grey)
- **Active** — checkbox checked, button enabled (coral-red)
- **Submitting** — button disabled (prevent double-submit), no spinner needed
- **Error** — error toast visible below button, modal still open, checkbox remains checked

**Layout rules:**
- Overlay: `position: fixed`, full viewport, `background: rgba(0,0,0,0.5)`, `z-index: 100`
- Flex-centred: `display: flex; align-items: center; justify-content: center`
- Card: `max-width: 480px; width: 90%; max-height: 90vh; overflow-y: auto`
- No dismiss on backdrop click (no onClick handler on overlay)
- No Escape key dismiss

**Animations:** None specified for this screen (legal/trust context, not brand screen)

---

### Checkbox (consent)
Standard HTML checkbox with brand purple accent when checked.

**States:**
- Unchecked (default, required by DPDP Act)
- Checked — accent colour `#9333EA`
- Hit target: wrapper `min-height: 48px; display: flex; align-items: flex-start; gap: 12px`

---

### Primary CTA Button ("I Agree & Continue")
Pill-shaped button, full width within card.

**States:**
- Disabled (unchecked): `background: #D1D5DB; color: #fff; opacity: 0.6; cursor: not-allowed`
- Active (checked): `background: #F43F5E; color: #fff; opacity: 1`
- Base: `height: 48px; border-radius: 24px; border: none; width: 100%; font-family: Nunito; font-weight: 600; font-size: 16px`

---

### Error Toast (inline, not floating)
Inline alert below button. Appears on Firestore write failure.

- `background: #FEE2E2; color: #991B1B; border-radius: 8px; padding: 12px 16px`
- `font-family: Nunito; font-size: 14px`
- `role="alert"` for accessibility
- No auto-dismiss in V1 (stays until user retries or page refreshes)

## 2. Relevant User Flows

### Flow 1: First-Time Onboarding — Steps 2 (Consent Modal)
Source: Design.md §4 Flow 1

- **Entry:** Sign In complete → `consentGiven: false` → ConsentModal renders
- **Step 1:** Modal appears as full-screen overlay (non-dismissable)
- **Step 2:** Parent reads data disclosure. Checkbox unchecked. Button disabled.
- **Step 3:** Parent ticks checkbox → button activates (coral-red)
- **Step 4:** Parent taps "I Agree & Continue" → write to Firestore → navigate to /profile
- **Success state:** Firestore `consentGiven: true`, `consentTimestamp: serverTimestamp()` written. Navigate to Child Profile Setup.
- **Error state:** Firestore write fails → toast "Something went wrong. Please try again." Modal stays open. No navigation.

## 4. Interaction Rules

| Rule | Value | Applies to |
|---|---|---|
| No dismiss on backdrop click | Omit onClick on overlay div | ConsentModal overlay |
| No dismiss on Escape | No keydown handler | ConsentModal |
| Button state tied to checkbox | `disabled={!checked \|\| submitting}` | CTA button |
| Checkbox must start unchecked | `useState(false)`, no `defaultChecked` | Checkbox |
| serverTimestamp required | `serverTimestamp()` not `new Date()` | recordConsent() Firestore write |
| Privacy Policy link: new tab | `target="_blank" rel="noopener noreferrer"` | Privacy Policy anchor |
| Error recovery: keep modal open | Do not navigate on catch | Submit handler |
| No creator photo | This is a legal screen, not a brand screen | ConsentModal |
| Responsive | `max-width: 480px; width: 90%` on card | ConsentModal card |
| No horizontal scroll < 480px | `overflow: hidden` on root, `box-sizing: border-box` | ConsentModal |
