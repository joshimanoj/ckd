# UAT Checklist: Story #6 — Admin Panel, Video Management

Generated by /prd | Sources: story-6.md (acceptance criteria, user flow), story-6-ui-context.md

## Setup

Platform: Web — open local dev server: http://localhost:5173
Test data: Sign in with the Firebase account that has `admin: true` custom claim set
Start at: http://localhost:5173/admin

Pre-requisite: Firebase custom claim `admin: true` must be set on your Google account
(one-time setup via Firebase Admin SDK CLI in Sprint 0).

---

## Checklist

### Flow 1: Access Control

[ ] 1. Open an incognito window → navigate to http://localhost:5173/admin
[ ] 2. Verify: redirected to http://localhost:5173/ (sign-in page), no error message shown
[ ] 3. Sign in with a non-admin Google account → navigate to /admin
[ ] 4. Verify: redirected to http://localhost:5173/, no "Access Denied" error exposed
[ ] 5. Sign in with the admin Google account → navigate to /admin
[ ] 6. Verify: Admin panel renders with sidebar and "Videos" content area visible

### Flow 2: Video List

[ ] 1. On the Admin panel, verify the video list loads automatically
[ ] 2. Verify: both active and inactive videos are shown (inactive rows visually dimmed)
[ ] 3. Verify: videos are ordered newest first (check publishedAt dates)
[ ] 4. Verify table columns: Title | Category | Published | Active

### Flow 3: Add Video — Happy Path

[ ] 1. Click "Add Video" button (coral/red, top right of content area)
[ ] 2. Verify: slide-over form opens from the right, 480px wide
[ ] 3. Fill in:
       - YouTube Video ID: `dQw4w9WgXcQ`
       - Tab away from the field
[ ] 4. Verify: Thumbnail URL auto-filled with `https://img.youtube.com/vi/dQw4w9WgXcQ/hqdefault.jpg`
[ ] 5. Verify: thumbnail preview image appears inline
[ ] 6. Fill in Title: `Test UAT Video`
[ ] 7. Select Category: `Rhymes`
[ ] 8. Fill in Duration: `3:30`
[ ] 9. Click "Publish"
[ ] 10. Verify: success toast "Video published successfully" slides up from bottom
[ ] 11. Verify: form closes automatically
[ ] 12. Verify: new video appears in the list at the top (newest first)

### Flow 4: Add Video — Validation Errors

[ ] 1. Click "Add Video"
[ ] 2. Click "Publish" immediately (all fields empty)
[ ] 3. Verify: inline error under YouTube Video ID field
[ ] 4. Verify: inline error under Title field
[ ] 5. Verify: inline error under Duration field
[ ] 6. Type `badformat` in Duration field → click elsewhere (blur)
[ ] 7. Verify: error "Use mm:ss format (e.g. 3:45)" shown inline
[ ] 8. Fix Duration to `2:30` → click elsewhere
[ ] 9. Verify: duration error disappears
[ ] 10. Close the form (do not publish)

### Flow 5: isActive Toggle

[ ] 1. Find a video in the list
[ ] 2. Toggle the isActive switch to OFF
[ ] 3. Verify: toggle visually turns off (grey)
[ ] 4. Open the Video Library in another tab (http://localhost:5173/library)
[ ] 5. Verify: the video is no longer visible in the library (requires refresh/re-fetch)
[ ] 6. Return to Admin panel → toggle the same video back to ON
[ ] 7. Verify: toggle visually turns on (purple)
[ ] 8. Refresh the library → verify video reappears

### Flow 6: Send Notification

[ ] 1. Scroll down to the "Send Notification" section (light purple background card)
[ ] 2. Verify: Title field pre-filled with "New rhyme added!"
[ ] 3. Fill in Body: `Check out our latest video!`
[ ] 4. Optionally fill in YouTube Video ID: `dQw4w9WgXcQ`
[ ] 5. Click "Send"
[ ] 6. Verify: status updates to "Sending..." immediately
[ ] 7. Wait for Cloud Function to process (or verify status in Firebase Console)
[ ] 8. Verify: status eventually shows "Sent" in green (or "Failed" in red if Cloud Function not deployed)

### Visual Check

[ ] Sidebar: dark indigo (#1E1B4B) background, white navigation links
[ ] "Add Video" button: coral/red, pill shape, white text
[ ] VideoForm: right-side panel, white/off-white background, purple focus rings on inputs
[ ] isActive toggles: purple when ON, grey when OFF
[ ] Notification panel: distinct light purple (#F3E8FF) background section
[ ] Error text: red, small font, inline below fields
[ ] Success toast: green, slides up from bottom, auto-dismisses

### Responsive Check

[ ] Desktop (> 768px): sidebar visible on left, content on right — correct two-column layout
[ ] Tablet/Mobile (480–768px): sidebar collapses or moves to top, content is full-width, no elements cut off
[ ] Minimum (375px): page scrollable, no elements overflowing horizontally

### Edge Cases

[ ] Publish button is disabled (greyed out) during Firestore write — cannot double-tap to duplicate
[ ] Form stays open if Firestore write fails (simulate by disconnecting from network mid-publish)
[ ] Very long video title (80+ chars): title field stops accepting input at 80 characters
[ ] Duration `0:00` accepted as valid (edge: a 0-second video)
[ ] Duplicate YouTube Video ID: no error shown (V1 — admin responsibility per story-6.md)
