## 2. Relevant User Flows

### Flow 1 — First-Time Onboarding (Steps 3–4 only)

Source: Design.md §4 Flow 1

**Step 3 — Child Profile Setup:**
- Parent enters child's name → selects age range (Under 3 / 3–4 / 5–6) → taps "Start Watching"
- Success state: profile written to Firestore, navigate to Video Library
- Error state: Firestore write fails → toast error, stay on screen, retry allowed

**Step 4 — Video Library:**
- Destination after profile creation
- Currently a placeholder (`data-testid="library-screen"`)

**Back navigation rule:** If user is on `/profile` and has no child profile, back button keeps them on `/profile` (not consent, not sign-in). `useAuth` routing logic enforces this — no manual handling needed in the component.

## 4. Interaction Rules

| Rule | Detail | Applies to |
|---|---|---|
| Form: inline validation on blur | Show error below name field only after user has blurred the field with invalid value | Name input |
| Button disabled state | `disabled` attribute + `aria-disabled="true"` when `name.trim().length === 0 \|\| selectedAge === null` | CTA button |
| Age pills: single select | Clicking a pill deselects all others; `aria-pressed` updated accordingly | Age pill group |
| Name max length | Silently trim to 50 characters — no error shown | Name input |
| Saving state | Button label → "Saving...", button disabled, no double-submit possible | CTA button |
| Error recovery | After toast shown, button re-enabled so user can retry | CTA button + error toast |
| Keyboard submit | `<form onSubmit>` handles Enter key — no separate keydown handler needed | Form |
| No scroll | All elements must fit on screen without scroll (max-width: 480px form, 8dp grid spacing) | Full screen |

## 5. Responsive Rules

- Primary target: 480–768px (mobile web)
- `max-width: 480px` form container, centred
- Age pills: `flex-wrap: wrap` so they wrap at very narrow widths (< 320px) without causing horizontal scroll
- `overflow-x: hidden` on root container
- At `< 480px`: reduce outer padding from 16px to 12px
- No horizontal scroll at any viewport width ≥ 320px
