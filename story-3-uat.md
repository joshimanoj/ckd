# UAT Checklist: Story #3 — Child Profile Setup

Generated by /prd | Sources: story-3.md (acceptance criteria, user flows), story-3-ui-context.md

---

## Setup

**Platform:** Web — open local dev server: `http://localhost:5173`
**Test data:** Sign in with a Google account that has `consentGiven: true` but NO child profiles in Firestore
**Start at:** App root `/` — you should be auto-redirected to `/profile`

> **Returning user test:** To test the returning user flow, complete the form once, then sign out and sign back in — you should land on `/library` instead of `/profile`.

---

## Checklist

### Flow 1: First-time profile creation (happy path)

- [ ] 1. Open `http://localhost:5173` — confirm you are redirected to `/profile` automatically
- [ ] 2. Screen shows "Who's watching?" heading with creator avatar alongside it
- [ ] 3. Top navigation bar is visible at the top
- [ ] 4. Name input is present with placeholder "e.g. Arjun"
- [ ] 5. Three age pill buttons are visible: "Under 3", "3–4 years", "5–6 years"
- [ ] 6. "Start Watching" button is visible but **disabled** (greyed out)
- [ ] 7. Type a child's name (e.g. "Arjun") — button is still disabled (age not selected yet)
- [ ] 8. Select "3–4 years" pill — pill highlights in purple, button becomes active (coral-red)
- [ ] 9. Tap "Start Watching" — button shows "Saving..." briefly
- [ ] 10. App navigates to `/library` (Video Library screen)
- [ ] 11. Check Firebase console / emulator: `users/{uid}/childProfiles` has one document with `name: "Arjun"`, `dateOfBirth`, and `createdAt`

### Flow 2: Age range single select

- [ ] 1. Navigate back to `/profile` (sign out and in again with no-profile account, or use emulator reset)
- [ ] 2. Click "Under 3" — only "Under 3" is highlighted
- [ ] 3. Click "3–4 years" — "Under 3" deselects, "3–4 years" highlights
- [ ] 4. Click "5–6 years" — "3–4 years" deselects, "5–6 years" highlights
- [ ] 5. Confirm: exactly one pill is highlighted at a time

### Flow 3: Name validation

- [ ] 1. Type only spaces in the name field — "Start Watching" stays disabled
- [ ] 2. Clear spaces, type "Arjun" — button still disabled (no age selected)
- [ ] 3. Select an age — button becomes active
- [ ] 4. Clear the name — button becomes disabled again

### Flow 4: Returning user skips profile screen

- [ ] 1. After creating a profile in Flow 1, sign out and sign back in
- [ ] 2. Confirm: app goes directly to `/library`, NOT `/profile`

## Visual Check

- [ ] Screen background is light purple (`#F3E8FF`) — not white
- [ ] "Who's watching?" is in Baloo 2 font (rounded, bold), deep indigo colour
- [ ] Creator avatar appears circular (40dp) next to the heading
- [ ] Name input has a rounded border (16dp radius); purple border on focus
- [ ] Unselected age pills: light purple background, purple text and border
- [ ] Selected age pill: solid purple background, white text
- [ ] "Start Watching" button: coral-red when active, grey when disabled, full width, pill shape
- [ ] Top navigation bar shows "Choti Ki Duniya" in purple
- [ ] Error toast (if visible): red text on light red background

---

## Responsive / Device Check

- [ ] At 480px width: all elements fit without horizontal scroll
- [ ] At 375px width: all elements fit, age pills wrap if needed, no overflow
- [ ] At 768px width: form remains centred, max-width respected
- [ ] Name input and age pills are at least 48dp tall (easily tappable on touch screen)
- [ ] "Start Watching" button is full-width and at least 48dp tall

---

## Edge Cases

- [ ] Type a very long name (50+ characters) — input silently caps at 50, no error shown
- [ ] Submit form → simulate network off before tapping — error toast appears with "Couldn't save profile. Try again.", URL stays at `/profile`, button re-enables
- [ ] Press Enter inside the name input (with age selected) — form submits correctly
